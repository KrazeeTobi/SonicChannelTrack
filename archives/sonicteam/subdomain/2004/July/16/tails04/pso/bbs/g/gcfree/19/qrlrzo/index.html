たなとす <bmlbrgjffb> 2002/10/03 19:37:45
かなり遅いレスですがまだ見てますか？
同じ大田ケーブルネットワーク利用者です
（ってことはご近所サンですね^^;）。

　家に帰ってきたばかりだしBBAがまだ届いてないので
GCでの接続はまだ試してないんですが、GCでもDCでも
ネットワークへの接続の部分は基本的に同じだと思う
のでお答えします。

　ケーブルモデムからGC（＋BBA）に直接繋いでいる
ようなので、問題は以下の2つあたりかなと思います。

1) IPアドレスが借りっぱなしになっている
2) MACアドレスの登録変更されてない

2)の方はご存知のようなので1)を確認した方が良いですね。
DHCPというのは接続した時にIP（ネットワーク上の住所みたい
なもの）を自動的に割り当ててくれるサーバです。普通は
そこから一時的にIPを借りてネットワークに接続するのですが、
この割当て数というのがケーブルモデムごとに決まっていて
通常1つだけです。
　もし、GCを接続する前にパソコンをネットワークに繋いで
いたりすると、そのパソコンがすでにIPを借りてしまっていて
それを返すまでは新しいIPを貸してもらえません。これがIPを
取得できない原因だと思います。
　Windowsは接続を切っても自動的にIPを返してくれないし、
貸出し期限はかなり長い（24時間とか）ので待っていても自動で
解放されません。そこで自分から「パソコンが借りていたIPを
返す」作業が必要です。

　作業はパソコンをケーブルモデムに繋いだ状態で、パソコン上
で行います。ちょっとうろ覚えなところがあって申し訳ないですが、
たしかスタートメニューから「ファイル名を指定して実行」を選んで
「winipcfg」を実行、そして「すべて解放」もしくはどのネット
ワークアダプタなのかわかっていればそれを指定して「解放」
を選べば解放されます。IP自動設定アドレスの欄が「0.0.0.0」に
なっていればOK。解放した後にLANケーブルをGCに繋ぎ代えて
ください。

　ケーブルモデムを再起動した程度ではIPは解放されないので、

[☆☆追加]BBAの動作確認方法
# 実はDC版の焼き直しだったり(^_^;

Windows 機の場合

PC(Win機)を LAN ケーブルを繋がずに起動する

(MS-DOS)コマンドプロンプトで「ipconfig」か「winipcfg」で
BBA と接続する LAN アダプタの IP アドレスを確認
# 大体169.254.???.???
# 「インターネット接続の共有」使用時は普通は「192.168.0.1」
その値に±１等して GC に設定する

GCとPCをクロスLANケーブル(かHUBとストレートケーブル２本)で繋ぐ
GCでリンクをクリックするなどして接続動作をさせる
(MS-DOS)コマンドプロンプトで「ping 先程 GC に設定した値」
通れば動作が確認される
(MS-DOS)コマンドプロンプトで「arp -a」
BBA の MAC アドレス(GC側にシールで貼付)が出れば
ちゃんと BbA に ping 出来ているか確認できる

他の OS でもやることは一緒

尚、ケーブルが抜けているとちゃんと警告が出ます

以下、追加分

「クロスLANケーブル」もしくは「HUBとストレートLANケーブル２本」
はありますか？

無ければ、チェックできません

上記のモノがある場合、
LANケーブルを繋がずにWindowsを起動
winipcfgでIPアドレスとサブネットマスクを確認
(IPアドレス:169.254.123.45、サブネットマスク:25.255.0.0とか)
PSOの「オプション」＞「ネットワークの設定」＞「プロバイダ設定」
＞「ユーザプロバイダ」で「はい」。後のも「はい」
「ユ−ザプロバイダ設定」を選択
(この行、任意)「プロバイダ５」を選択して「BBA確認用」と入れる
「接続設定」で２番目の「自分で設定する」を選択
例えば
IPアドレス:169.254.123.46
サブネットマスク:255.255.0.0
デフォルトルータ以下は空欄に

PCとGCを「クロスLANケーブル(か、ストレート２本とHUB)」で接続
HUBを使う場合は両方とも普通のポートに接続します
カスケードとかアップリンクポートでは駄目です

(MS-DOS)コマンドプロンプトで
「ping 169.254.123.46(上記例)」
「Time out」とか出なければ成功

もう一つ確認するなら、同じく「arp -a」
GCに設定したIPの隣に出てきた値と、
BBAのGC側のシールの値を比べて一致すればOK

確認が終わったら、プロバイダ４(自分の設定)に戻すのを忘れずに

更に追加
WinMEは手元にないので
・とりあえずLANケーブルを挿してみる
駄目だったら
「コントロールパネル」＞「ネットワーク」を開く
自分のLANアダプタの「TCP/IP -> なんとか」となっているのを開く
IPアドレスを指定で「192.168.0.1」
サブネットマスクは「255.255.255.0」
GCの方は
IPアドレスを指定で「192.168.0.2」
サブネットマスクは「255.255.255.0」
でLANケーブルで繋いで
「ping 192.168.0.2」

終了後パソコンの設定を戻すのを忘れずに
